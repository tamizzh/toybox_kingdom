# Party Pals Arena — Store Submission Checklist

What's already done in the project vs. what still needs a device, a developer account, or a Mac.
Everything below the "In the project" sections requires tooling that can't run from this repo alone.

---

## ✅ Already in the project
- App name, version (`1.0.0`), description, branded `icon.svg`, boot splash (`assets/splash.png` + bg color) — `project.godot`.
- Landscape lock, mobile renderer, responsive camera + `canvas_items` stretch.
- Audio (music + SFX, generated by `tools/gen_audio.py`), volume settings persisted (`SaveManager`).
- Monetization scaffolding: `MonetizationManager` (ads/IAP facade, **simulated** until plugins added),
  Shop (cosmetic packs + Remove Ads), Settings (audio/restore/privacy/credits), first-run consent gate.
- `export_presets.cfg` with **Android (AAB)** and **iOS** presets, bundle id `com.partypals.arena`.
- Curated 12-game launch roster (the rest stay in-repo for content updates).

---

## 1. Accounts & tools (one-time)
- [ ] **Google Play Console** account ($25 once).
- [ ] **Apple Developer Program** ($99/yr) — required for any iOS build/submit.
- [ ] A **Mac with Xcode** — Godot's iOS export *cannot* run on Windows. Everything else here works on this PC.
- [ ] **Hosted privacy policy URL** (required because the app shows ads). Put the URL in:
      - `ui/settings_screen.gd` → `PRIVACY_URL`
      - Play Data Safety form + App Store privacy questions.

## 2. Godot export setup (this PC, for Android)
- [ ] Editor → Manage Export Templates → download **4.6.2-stable** templates.
- [ ] Install Android **SDK + JDK 17**, set paths in Editor Settings → Export → Android.
- [ ] Project → Install Android Build Template (enables the Gradle build needed by ad/IAP plugins).
- [ ] Generate a **release keystore**; set it in the Android preset (keep it backed up — losing it blocks updates).
- [ ] Export the AAB:  `godot --headless --export-release "Android" build/android/party_pals_arena.aab`

## 3. Ads + IAP plugins (the real-SDK wiring)
- [ ] Add a Godot 4.6-compatible **AdMob** plugin (e.g. `cengiz-pz/godot-admob-android` + `-ios`) under `addons/`.
- [ ] Add **Play Billing** (Android) and an **iOS IAP** plugin under `addons/`.
- [ ] Fill the `TODO(real-sdk)` blocks in `core/monetization_manager.gd`:
      init, consent (UMP/ATT), interstitial, rewarded, purchase, restore.
- [ ] Create these product IDs in **both** stores (must match `theme/cosmetics.gd` / shop):
      `remove_ads`, `pack_neon`, `pack_pastel`, `pack_candy`.
- [ ] Put your real **AdMob app id** (AndroidManifest / Info.plist) and **ad unit ids** in the plugin config.
- [ ] Test in **sandbox**: consent shows once, rewarded grants, interstitial only between matches & frequency-capped,
      purchases persist across relaunch, Restore works.

## 4. Store listing assets
- [ ] Screenshots — use `tools/shot_game.tscn` (e.g. `-- snake_battle 2` / `-- sprint_race 2 3 win`) at 1080p+.
      Capture 4–8 across different games for each store.
- [ ] **Feature graphic** (Play, 1024×500) and any required preview art.
- [ ] Short + full description, category (Casual/Arcade), content rating questionnaire (IARC).
- [ ] Play **Data Safety** form + Apple **privacy nutrition labels** (declare ads + device identifiers).

## 5. iOS (on the Mac)
- [ ] Open the exported Xcode project; set the signing **Team**, provisioning profile, bundle id.
- [ ] Add launcher icon set + launch screen, archive, upload via Xcode/Transporter to App Store Connect.

---

## Notes
- Audio & art are **original/generated** (no third-party license obligations).
- Keep `com.partypals.arena` consistent across `export_presets.cfg`, both stores, and plugin configs.
- Until the ad/IAP plugins are installed, the game runs fully with **simulated** purchases/ads (safe for dev/testing).
